Overview
========

Proprietary Notice: This document contains only non-proprietary
information.

The purpose of the MPW is to identify the Units of Replaceability (UoRs)
composing an OMS Mission Package during different phases of program
development. OMS defines UoRs as the ‘building blocks’ of a Mission
Package that can be separately and independently replaced and/or
upgraded, specifically to include the Mission Package itself and its
constituent Platform, Subsystems, Services, and Isolators. The MPW
should identify each OMS Service, Subsystem, and Isolator in a Mission
Package. Each completed row of the MPW identifies one of the Mission
Package’s OMS UoRs. Table 1.0-1 depicts the MPW.

When an Adopting Program (AP) decides to incorporate a compliant OMS
Mission Package into a weapon system, the MPW is used to document the
planned UoRs of that Mission Package. MPWs can be used to document UoRs
both within a specific Mission Package as well as across a portfolio of
systems with OMS Mission Packages. Therefore stakeholders of a MPW can
include both those supporting the Adopting Program (AP) of a specific
OMS system/Mission Package as well as higher level stakeholders with
goals and concern spanning multiple OMS systems. Design decisions
documented in the MPW should reflect the goals of all applicable
stakeholders. The information specified by the MPW template may also be
included as part of an acquisition document (Request for Information,
Request for Proposal, or a proposal).

Initially the MPW documents planned and/or assumed details of the
Mission Package, and as the design, implementation, and integration of
the Mission package evolves, the MPW is updated. The MPW allows the AP
to continuously assess Mission Package content and determine if the
Mission Package is meeting the intent and requirements of the AP.

Speculative MPWs can be used to document and evaluate proposed changes
to a Mission Package, and earlier versions of the MPW can be used to
document the evolution of a Mission Package across its life cycle.

When a Mission Package is completed, the MPW serves as a standardized
top-level summary of the Mission Package composition and is a key
artifact used to assess the OMS compliance of the Mission Package and
its Platform, Subsystems, and Services. The MPW identifies the
compliance Tier for each of the applicable UoRs in the Mission Package,
and identifies which Mission Package Service Capabilities and Subsystems
are on the ‘Lists for Compliance’ defining compliance requirements for
compliance tiers 1 and 2. Note that a compliance tier is not identified
for an Isolator, as Isolators take on the compliance tier of the Mission
Package.

Finally, the MPW identifies the Cyber Survivability Risk Category (CSRC)
of each Mission Package.
